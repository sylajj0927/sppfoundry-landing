## 本輪完成

### 1. 修正 BOOM / BiteMap 外部連結箭頭 emoji fallback

- 修正 BOOM / BiteMap 外部連結箭頭在 iPhone 顯示成藍色 emoji 的問題。
- 將文字 `↗` 改為 inline SVG external-link icon。
- SVG 使用 `currentColor`，維持既有 `.arrow` wrapper 作為尺寸、顏色與對齊控制。
- 保留 AICert 站內 `→`，不擴大修改範圍。

### 2. 修正 mobile section h2 被 cut

- 修正 mobile section h2 中文長句右側被裁切的問題。
- 在 mobile 下為 `section h2` 放寬中文換行策略：
  - `max-width: 100%`
  - `word-break: normal`
  - `overflow-wrap: anywhere`
  - `line-break: loose`
- 保留 Hero H1 mobile 專用分行，不影響 Hero。

### 3. 修正 iPhone Safari full-page screenshot 黑底

- 修正 iPhone Safari full-page screenshot 中 Hero 下方大面積黑底問題。
- mobile 關閉 `body::before` 背景紋理層，避免 full-page screenshot 對 `position: fixed`、`z-index: -1`、`mask-image` 的不穩定處理。
- mobile 為 `main` / `footer` 補 `background: var(--cream)`，讓長截圖透明區域有穩定 light background。
- mobile 設定 `footer { margin-top: 0; }`，消除 footer 上方 margin 露出的黑帶。

### 4. 收斂 mobile h2 字級

- 收斂 mobile section heading 視覺重量，避免 h2 比 Hero H1 更搶眼。
- `h2` mobile font-size：
  - `clamp(27px, 8.2vw, 34px)` → `clamp(24px, 7.2vw, 29px)`
- `h2` mobile line-height：
  - `1.34` → `1.38`
- 不改 Hero H1、不改 desktop h2、不改文案與資訊架構。

### 5. Production 驗證通過

- BOOM / BiteMap 外部箭頭正常，不再 emoji 化。
- mobile section 標題完整，不再被裁切。
- iPhone Safari full-page screenshot 背景為米白，沒有黑底。
- footer 上方黑帶消失。
- Hero H1 / section h2 比例順眼。
- desktop 沒有跑版。
