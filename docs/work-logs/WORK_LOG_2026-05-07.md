### 3. 外部連結箭頭標準化

- `.arrow { font-variant-emoji: text; }` 新增一行 CSS
- 目標：BOOM / BiteMap 的 `↗` 在 iOS / Android 手機不被渲染成彩色 emoji 圖示
- 維持文字符號的專業感，不更換箭頭符號

### 4. 手機版 Horizontal Overflow 修正

- 在 `@media (max-width: 920px)` 內新增 `.lab-feature::after, .hero-card::before { display: none; }`
- 修正根本原因：iOS Safari 對 CSS Grid item 的 `overflow: hidden` 無法可靠裁切 `position: absolute` 的 pseudo-elements，導致 `.lab-feature::after { right: -76px }` 等出血裝飾造成頁面右側被撐大
- 精準修正，不使用 `html / body overflow-x: hidden`
- 桌機版裝飾 pseudo-elements 完全不受影響

## Hero 視覺方向試驗與回退

### 試驗內容

本輪參考編輯感設計語言（白底 / 大量留白 / 黑色主標 / 紅色強調 / 藍色 CTA / 紅色細曲線）嘗試 Hero prototype：

- h1 關鍵詞「可以開始」加紅色 `<span class="hero-em">`
- Hero primary CTA 改藍底白字（`--hero-blue: #1a4ed8`）
- Hero secondary CTA 改 outline 矩形感（`border-radius: 6px`）
- 新增紅色 S 形曲線 SVG 元素（`class="hero-curve"`）

### 回退決策

本機檢查後判斷參考設計語言不適合直接套用在目前 Hero，原因：

- Hero card 維持原暖色系設計語言，紅藍曲線放在中間造成左右視覺語言不一致
- 矩形藍色 CTA 與現有 pill 型按鈕系統落差較大，需要整體 CTA 系統一起改才合理
- 紅色 S 形曲線在暖色背景底下吃力

**已完全回退 Hero prototype 殘留**（包含 CSS block、HTML class、SVG 元素），Hero 維持原穩定版本。
