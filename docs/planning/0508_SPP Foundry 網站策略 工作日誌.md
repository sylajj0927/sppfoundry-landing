### **🎯 核心策略結論(下次開新對話必讀)**

**對外的一句話:**

**Phase 1(現在 → 累積 3 個案例):**

* TA:一人工作室、療癒師、身心靈、篆藝、手作

### **🏆 競品分析:SIBYL.DESIGN**

#### **它是誰**

「闆娘背景的客製化 GAS 系統,賣給小型實體店」。15 年闆娘經驗,案例已累積 5+ 個(編號到 CASE 05),產品線包括預約管家、點餐管家 Pro、下單管家、行政流程管家。

#### **它最強的 8 個地方(Sylvia 該學的觀念,不是表面)**

1. 創辦人故事 \= USP(15 年闆娘背景)  
2. 比較表(市售 vs 自己)幫客戶做決策  
3. **公開維護報價**(800/500/1500-2500)  
4. 案例編號暗示累積感  
5. **痛點 → 解法 → demo 案例結構**(最該學)  
6. LINE「需求代碼 1-5」降低開口門檻  
7. **開工三守則**(合作邊界)  
8. package 化 \+ 行業標籤

#### **它的 8 個弱點(Sylvia 該避免的)**

1. 過度依賴「闆娘」身份,擴展性低  
2. 服務方法論薄弱,只能展示結果不能展示過程  
3. **用 Google Sites 架站,SEO 為零**(Sylvia 的真實 HTML 站完勝)  
4. 過度依賴 LINE,沒有非同步入口  
5. 技術深度沒有顯式露出  
6. 沒有「不適合誰」邊界  
7. 視覺有點過載(深色 \+ 高飽和橙 \+ 大量 emoji)  
8. 創辦人不露臉

#### **Sylvia vs SIBYL 的真正差異**

| 比較點 | SIBYL | SPP Foundry |
| ----- | ----- | ----- |
| 背景 | 15 年闆娘 | 15 年數位行銷/品牌/銀行 \+ Vibe Coding |
| 後端架構 | GAS \+ Sheet 直接讀寫 | **Cloud Run × 5 \+ SWR 雙層快取 \+ 降級設計 \+ 可觀測性**(企業級) |
| SEO/GEO | 零 | 強項 |
| TA | 闆娘(實體店) | Phase 1 療癒師,Phase 2 品牌主 |
| 視覺調性 | 深色 \+ 橙色 \+ emoji | Linear / Notion / Vercel(克制專業) |

**核心判斷:** 不要打 SIBYL,因為朋友介紹的客戶不會去比價。**用顱薦案例的技術深度展現自己的層級。**
